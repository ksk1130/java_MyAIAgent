# MyAIAgent 将来の機能拡張案 (Future Enhancements)

現在の `ChatCLI` （ツールの利用とステップループ実行）を基盤に、より自律的で高度な「Agentic AI」へと進化させるための拡張アイデアです。

## 1. 「思考ループ」と「計画 (Planning)」の分離
**課題:**
現状は、ユーザーの命令に対してAIが「ツールを使うか、そのまま答えるか」を場当たり的に（1ステップずつ）判断しています。

**解決策 (ReAct / Plan-and-Solve):**
ツールを実行する前に、明示的な「思考・計画フェーズ」を設けます。
- LLMに「タスク解決のためのステップごとのTo-Doリスト」を作成させます。
- メインループがそのTo-Doリストをシーケンシャルに消化し、必要に応じて計画を動的に修正するアーキテクチャへの移行。

## 2. 高度なコード編集ツールの導入 (AST / Diffベース) ✅ 完了
**課題:**
現在の `FileWriterTool` はファイル全体の上書き保存を行うため、長大なソースコードに対してはコンテキストウィンドウのリソース（出力時間・トークン）を無駄に消費します。

**解決策:**
ファイルの指定行のみをピンポイントで書き換えるツール（`FileEditorTool`）を実装しました。これにより数千行のファイルでも、必要な数行の変更を迅速かつ安全に行うことができます。

## 3. メモリの永続化とナレッジベース (RAG) の導入
**課題:**
現行の `MessageWindowChatMemory` はオンメモリ管理であり、アプリケーションを終了すると過去の文脈や学習内容を失ってしまいます。

**解決策:**
- **短期記憶:** チャットの履歴を SQLite などのローカルDBやファイルに永続化し、次回の起動時にも会話の文脈を引き継げるようにします。
- **長期記憶 (Knowledge Base):** プロジェクト固有のコーディング規約や設計情報などを「知識アイテム」としてベクトルDB（Chroma など LangChain4j 対応のストア）へ保存し、類似タスク時に自動でRAG（検索拡張生成）させる仕組みを追加します。

## 4. ブラウザ操作 / Web検索の自律化
**課題:**
エラー発生時や最新のライブラリ仕様を確認する際、現在はユーザー自身が検索して情報を与える必要があります。

**解決策:**
- **Web検索:** `WebSearchTool`（Tavily API や Google Search API を利用）を追加し、自律的にドキュメントを検索させます。
- **DOM解析:** Playwright などのブラウザ自動化ツールを結合し、ローカル環境で立ち上げたWebアプリのUIを直接AIに視認・操作させるサブエージェントを追加します。

## 5. 自律的なエラーリカバリ (Self-Correction)
**課題:**
現在のエージェントループは、ローカルコマンドが失敗した際に「エラーが出ました」とユーザーにエラーメッセージを返すだけになることが多いです。

**解決策:**
ステップループ内に「自己修復 (Self-Reflection)」の機構を組み込みます。ツール実行結果がエラーだった場合、直ちにユーザーへ報告するのではなく、AI自身にエラー内容（スタックトレース等）を解析させ、コマンドやコードを修正して再実行（リトライ）するループを設けます。

---
**推奨される最初のステップ:**
実装コストに対して非常に費用対効果が高いのは **「2. 高度なコード編集ツールの導入」** と **「5. 自律的なエラーリカバリ」** です。この2つが揃うだけで、バグ修正やリファクタリングを完結できるレベルに到達します。
